## 1. VISÃO GERAL DO PRODUTO

### 1.1 O que é o P.I.T?
O P.I.T (Performance · Intelligence · Tracking) é uma plataforma web de gestão competitiva para a comunidade de FIFA Pro Clubs 11v11. Integra-se à API da EA Sports para capturar dados, oferecendo: Discovery automático de times, estatísticas detalhadas, matchmaking, sistema de rating (Pit Rating™), campeonatos com anti-W.O., e Hall of Fame.

Atualmente o cenário brasileiro gerencia tudo de forma fragmentada: WhatsApp, planilhas, Discord. O P.I.T é a primeira plataforma profissional deste ecossistema.

**Stack Técnico:**
- Frontend: Next.js 14+ (App Router, TypeScript)
- Backend: Supabase (PostgreSQL 15+, Auth, RLS, Edge Functions, Realtime, Storage)
- Cron/Automação: n8n (VPS)
- Cookie Renewal: Puppeteer (VPS)
- Hosting: Vercel + VPS (Contabo/Hetzner)
- Pagamento: Mercado Pago (PIX automático)
- UI: Tailwind CSS + shadcn/ui + Lucide React

### 1.2 Problemas Resolvidos
- Stats se perdem (API retorna só últimos 10 jogos)
- Escalações via WhatsApp sem histórico
- Matchmaking manual e lento
- Disputas de W.O. sem dados concretos
- Sem classificação unificada entre comunidades
- Managers não sabem o clubId do próprio time
- Nomes de times com encoding quebrado na API (acentos)
- Não há registro histórico de conquistas (Hall of Fame)

### 1.3 Público-alvo
- Jogador: Jogador ativo de Pro Clubs 11v11. Necessidade: stats, evolução, comparações, ser descoberto.
- Manager: Gerencia um time competitivo (também é jogador). Necessidade: elenco, escalações, matchmaking, reivindicar time.
- Admin PIT: Administrador da plataforma (Wander). Necessidade: painel admin, validar reivindicações, Discovery, campeonatos.

## 2. SISTEMA DE PAPÉIS (ROLES)

O P.I.T usa um sistema de papéis acumulativos. Todo usuário é Jogador por padrão. Papéis adicionais são concedidos por ações específicas.

| Papel | Como Obtém | Acessa | Visão na Interface |
|---|---|---|---|
| Jogador | Automático ao cadastrar | Perfil pessoal, stats, histórico, escalações | "Meu Perfil" |
| Manager | Reivindica ou cria um time | Tudo de Jogador + gestão de time, elenco, matchmaking, escalação | "Meu Time" |

Alternador de contexto na sidebar: Manager vê (Meu Perfil | Meu Time). Moderador vê (+ Moderação). Admin vê (+ Painel Admin completo).

Visão exclusiva para Moderadores e Admin. Foco operacional:
- Campeonatos: Criar, gerenciar rodadas, encerrar campeonatos e corujões.
- Fila de Reivindicações: Claims pendentes com foto de prova. Aprovar/rejeitar.
- Disputas W.O.: Visualizar contestações, avaliar provas, emitir decisões.

O Admin PIT vê tudo da Moderação MAIS as áreas restritas:

- Assinaturas: Gerenciar planos Premium, cancelamentos, upgrades.
- Discovery: Disparar varreduras, inserir clubIds manualmente, monitorar base.
- Inserir ID Manual: Campo para clubId não encontrado. Puxa 10 jogos e confirma.

## 4. FONTE DE DADOS — API EA SPORTS

### 4.1 Endpoint
GET proclubs.ea.com/api/fc/clubs/matches?platform=common-gen5&clubIds={id}&maxResultCount=10&matchType=friendlyMatch

### 4.2 Dados Retornados
Jogo: matchId, timestamp, placar, clubIds e nomes de ambos os times.
Jogador: playername (gamertag), goals, assists, passes, tackles (tacklesmade, tackleattempts), rating, pos (goalkeeper/defender/midfielder/forward), shots, cards, cleansheetsgk, saves, minutesplayed, man_of_the_match.
Discovery: Cada resposta contém dados dos DOIS times + todos os jogadores. Uma consulta revela até 10 novos times.

### 4.3 Normalização de Nomes (Encoding Corrompido)
A API retorna nomes de times com encoding corrompido (Latin-1 interpretado como UTF-8). Caracteres afetados:

Solução técnica: Pipeline de normalização no momento da ingestão. Decodificar string como Latin-1 e re-encodar como UTF-8. Armazenar DOIS campos: ea_name_raw (original da API, para match exato) e display_name (normalizado, para exibição e busca fuzzy).

### 4.4 Proteção Anti-Bot (Cookie Akamai)
A API usa proteção Akamai Bot Manager (cookies ak_bmsc e bm_sv). Estratégia:
- Recomendada (MVP): Puppeteer na VPS — headless browser acessa o site EA, obtém cookies, passa para as requisições da API.
- Fallback: Browserless.io ou ScrapingBee (serviços externos que lidam com Akamai).
- Fallback 2: Proxy residencial rotativo.
- Renovação: Microserviço na VPS que renova cookies a cada 15-30 min e disponibiliza via endpoint interno.

### 4.5 Coleta Periódica
- Cron: Seg/ter/qui às 04:00 AM (n8n). Puppeteer renova cookies antes.
- Manual: Botão no painel Manager (rate limit: 1x/30min por time).
- Deduplicação: matchId como chave única (ea_match_id UNIQUE).
- Timestamp: Convertido de Unix epoch para UTC-3 (Brasília).

## 5. SISTEMA DE DISCOVERY (SNOWBALL) — PRÉ-MVP

O Discovery é a funcionalidade mais estratégica do P.I.T. Constrói a base de times exponencialmente.

### 5.1 Crescimento Exponencial

| Iteração | Consultados | Novos Descobertos | Total Base |
|---|---|---|---|
| 1ª | 1 (seed) | Até 10 | ~11 |
| 2ª | 10 | Até 100 | ~111 |
| 3ª | 100 | Até 1.000 | ~1.111 |
| 4ª+ | 1.000+ | Desacelera (repetições) | ~2.000+ |

Processamento em paralelo (10-20 reqs simultâneas) com delay entre batches. Meta: 500+ times antes do lançamento.

### 5.2 Dados Armazenados por Time Descoberto
ea_club_id, ea_name_raw (nome original API), display_name (nome normalizado), discovered_at, discovered_via (clubId que levou à descoberta), last_scanned_at, claimed_by, status (unclaimed/pending/active).

### 5.3 Time Não Encontrado na Base (Fluxo de Inserção Manual)
Se um manager busca seu time e não encontra:
1. Sistema exibe mensagem: "Time não encontrado. Sua solicitação foi enviada ao admin."
2. Notificação é enviada ao Admin PIT com o nome buscado.
3. Admin acessa o Painel Admin e busca o clubId em ferramenta externa.
4. Admin insere o clubId no campo "Inserir ID Manual" do painel.
5. Sistema puxa últimos 10 jogos, admin confirma, time é adicionado à base.
6. Manager é notificado e pode prosseguir com a reivindicação.

## 6. FLUXO DE REIVINDICAÇÃO DE TIME

### 6.1 Fluxo Completo
1. Manager se cadastra (e-mail + senha + gamertag EA)
2. Acessa "Vincular meu Time" e digita o nome do time
3. Sistema faz busca fuzzy na tabela discovered_clubs (usando pg_trgm no display_name)
4. Exibe resultados. Manager seleciona o time.
5. Sistema puxa últimos 10 jogos da API em tempo real e exibe (placares + adversários)
6. Manager confirma: "Esses são meus jogos?"
7. Se confirmar: sistema solicita FOTO DE PROVA mostrando que é manager do time dentro do jogo
8. Manager faz upload da foto. Status do time muda para "pending"
9. Admin PIT recebe notificação e acessa fila de reivindicações no Painel Admin
10. Admin avalia a foto e aprova ou rejeita
11. Aprovado: time muda para "active", manager é vinculado. Rejeitado: manager é notificado.

Proteção anti-fraude: Enquanto "pending", nenhum outro pode reivindicar. Se "active" (outro manager), aparece "Time já reivindicado" + opção de abrir disputa.

## 7. SISTEMA DE POSIÇÕES

### 7.1 Formação Base: 3-5-2

| Posição Detalhada | Sigla | API Retorna | Quantidade na 3-5-2 |
|---|---|---|---|
| Goleiro | GK | goalkeeper | 1 |
| Zagueiro | ZAG | defender | 3 |
| Volante | VOL | midfielder | 2 |
| Meia Central | MC | midfielder | 1 |
| Ala Esquerdo | AE | midfielder | 1 |
| Ala Direito | AD | midfielder | 1 |
| Atacante | ATA | forward | 2 |

### 7.2 Lógica de Mapeamento
Quando a API retorna categoria genérica, o sistema verifica posições cadastradas do jogador:
- Exemplo: Jogador cadastrou Principal: VOL, Secundária: ZAG. Se API retornar "defender", jogou como ZAG. Se "midfielder", jogou como VOL.
- Ambiguidade: Se API retornar "midfielder" para jogador com Principal: MC e Secundária: AE, sistema registra como posição principal (MC) por padrão. Manager pode corrigir na escalação.

## 10. CORUJÕES DIÁRIOS (MVP)

Corujões são torneios relâmpago mata-mata realizados todas as noites. São o principal motor de engajamento e receita recorrente.

### 10.1 Especificação

| Parâmetro | Valor |
|---|---|
| Formato | Mata-mata (eliminação simples) |
| Capacidade | 8 a 32 times (variável, definido pelo admin/moderador) |
| Fase de grupos | Opcional: 2 a 8 grupos (ex: 16 times = 4 grupos de 4, top 2 avançam) |
| Frequência | Diário (todos os dias da semana) |
| Horário | Início às 22:00. Duração até ~00:00-01:00 |
| Mínimo para acontecer | 8 times inscritos e confirmados |
| Inscrição | R$ 3,00 por time (via gateway de pagamento) |
| Chaveamento | Automático (sorteio ou seed por Pit Rating quando disponível) |
| Resultados | Coletados automaticamente via API EA após cada rodada |
| Premiação | Banner digital com MVPs, artilheiro, Pitbull etc. (Hall of Fame automático) |

### 10.2 Fluxo de Inscrição
1. Admin/Moderador cria o corujão do dia (define capacidade, horário, formato)
2. Corujão aparece na página "Corujões" com vagas disponíveis
3. Manager clica em "Inscrever meu time"
4. Sistema gera cobrança via gateway (Mercado Pago) — PIX automático de R$ 3,00
5. Se pagamento confirmado: time inscrito. Se sistema de confiança ativo: time inscrito com 24h para pagar.
6. Quando mínimo de 8 times é atingido: corujão é confirmado e todos são notificados
7. Se não atingir mínimo até o horário: corujão cancelado, pagamentos estornados automaticamente
8. Chaveamento gerado automaticamente. Confrontos publicados.

### 10.3 Sistema de Confiança para Pagamento
Para não travar a operação com pagamentos instantâneos, o sistema permite inscrição por confiança com prazo de 24 horas:

| Status | Regra | Ação |
|---|---|---|
| Confiável | Time sem pendências. Pode se inscrever com 24h após o corujão para pagar. | Joga normalmente |
| 1ª Inadimplência | Não pagou dentro de 24h. | Alerta |
| 2ª Inadimplência | Reincidiu no não-pagamento. | Suspensão 1 mês |
| 3ª Inadimplência | Terceira vez sem pagar. | Banido até próximo FIFA |

Regra de desbloqueio: Para sair de suspensão, o time precisa quitar TODAS as pendências + cumprir o período. O ban por 3ª inadimplência só é resetado com o lançamento do próximo FIFA.
Alternativa: Time pode optar por sempre pagar adiantado (PIX instantâneo) e nunca usar o sistema de confiança.

## 12. HALL OF FAME

Registro permanente de destaque por corujão e campeonato. Dados extraídos automaticamente da API. No MVP funciona para Corujões. Na Fase 2 expande para Campeonatos de Liga.

| Premiação | Critério | Campos da API |
|---|---|---|

| MVP da Final | Maior rating na partida final | rating |
| Artilheiro | Mais gols no campeonato (todas as fases) | goals (acumulado) |
| Garçom (Assistente) | Mais assistências no campeonato | assists (acumulado) |
| Pitbull 🐕 | Melhor marcação: tackles completados + tentados + rating defensivo | tacklesmade, tackleattempts, rating |
| Muralha | Goleiro com mais defesas e/ou clean sheets | saves, cleansheetsgk |
| Melhor Rating Médio | Maior rating médio do campeonato (mínimo X jogos) | rating (média) |

Persistência: Hall of Fame é permanente. Cada campeonato/corujão tem sua própria página de Hall of Fame.

## 13. PIT RATING™ (FASE 2)

REGRA FUNDAMENTAL: Só conta jogos que acontecem PELA plataforma (campeonatos ou matchmaking). Jogos externos não afetam rating.

- Rating inicial: 1500 pontos.
- Calibração: Primeiros 15 jogos com multiplicador 2x.
- Filtros: Competitivo (só campeonatos, padrão) e Geral (+amistosos PIT).
- Ligas automáticas:
  - Elite: 2200+ pontos
  - Ouro: 2000-2199
  - Prata: 1800-1999
  - Bronze: 1600-1799
  - Acesso: abaixo de 1600
- Soft reset: 10-15% compressão por temporada EA FC.

- payment_status: pending, paid, refunded, overdue, cancelled
- hall_of_fame_award: champion, mvp_final, top_scorer, top_assister, pitbull, muralha, best_avg_rating

### 15.3 RLS Policies (Row Level Security)
- Todas as tabelas com RLS habilitado
- Jogadores: ver próprias stats, não ver dados de outros sem permissão
- Manager: acessar dados do próprio time
- Moderador: acessar dados operacionais (claims, tournaments, disputes)
- Admin: acesso irrestrito via service_role key

### 15.4 Views Materializadas
- v_player_stats: agregado de stats do jogador (gols, assists, rating médio, partidas por temporada)
- v_club_stats: agregado do time (W/D/L, gols, players top)
- v_financial_dashboard: receita, inadimplências, projeções (admin only via service_role)
- v_club_rankings: ranking de times por performance e Pit Rating
