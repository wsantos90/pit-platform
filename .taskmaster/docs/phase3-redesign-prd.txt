# P.I.T Platform — Phase 3 Frontend Redesign PRD
# Version: 3.0 | Date: 2026-03-26 | Author: Claude (Anthropic)
# Status: Ready for TaskMaster parsing

## OVERVIEW

P.I.T (Performance · Intelligence · Tracking) is a competitive SaaS platform for FIFA Pro Clubs 11v11
management in Brazil. The backend is solid (54+ REST endpoints, 25+ Supabase tables, 5+ views, RLS,
MercadoPago PIX payments). The Phase 3 goal is a complete frontend redesign that makes the platform
feel alive, data-rich, and professional — like a real football analytics tool.

### Visual Identity (mandatory — do not deviate)
- Style: Dark Mode OLED + Data-Dense Dashboard + Minimalism
- Primary palette: OLED black (#0A0A0F base) + Orange/Amber (#F97316 primary)
- Design System MASTER.md: pit-platform/design-system/p.i.t/MASTER.md (source of truth for all visual specs)
- Typography: Russo One (display/headings), Inter (body), JetBrains Mono (data/numbers)
- No hardcoded colors — all via CSS custom properties
- No emojis as icons — use Lucide React exclusively
- All implementations must be dark-first, data-dense, esports-editorial tone

### Tech Stack
- Next.js 16 (App Router) + TypeScript 5 + Tailwind CSS 3 + shadcn/ui
- Supabase (PostgreSQL 15+, Auth, RLS, Realtime, Storage)
- MercadoPago (PIX payments, already integrated)
- Recharts (charts — radar, line, area, bar)
- @tanstack/react-table (data tables)
- Context switching: profile / team_id / moderation / admin (useContext hook)
- RBAC: player → manager → moderator → admin (cumulative roles)

### Definition of Done (per task)
1. Follows user story mapped to the task
2. Uses ONLY tokens from design-system/p.i.t/MASTER.md (no hardcoded colors)
3. All states covered: loading (skeleton), empty, error, success, no-permission
4. Dark OLED theme default — no light backgrounds
5. Responsive: mobile (375px), tablet (768px), desktop (1024px), wide (1440px)
6. `npm run build` passes with no TypeScript errors
7. `npm test` passes (if tests exist for the component)
8. Skills invoked: /frontend-design for implementation, /ui-ux-pro-max for review

## BACKEND GAPS — NEW ENDPOINTS NEEDED

The following endpoints do not exist and must be created alongside the frontend tasks that depend on them:

1. GET /api/rankings?type=player|club|position&period=week|month|season&limit=50
   - Server reads v_player_stats or v_club_stats with ORDER BY and LIMIT
   - No auth required (public)

2. GET /api/player/[id]/form?last=10
   - Returns last N match results for a player as W/D/L array
   - Used by FormStrip component

3. GET /api/club/[id]/form?last=10
   - Returns last N match results for a club as W/D/L array

4. GET /api/player/[id]/records
   - Returns personal bests: best rating, most goals in one match, longest win streak

5. GET /api/club/[id]/head-to-head/[otherId]
   - Returns H2H record between two clubs (wins/draws/losses + last 5 matches)

6. POST /api/upload/image
   - Validates and uploads to Supabase Storage
   - Body: FormData with file + bucket + path
   - Returns: { url: string }

Each endpoint must:
- Use createServerClient from @supabase/ssr
- Include proper error handling (400, 401, 404, 500)
- Return typed response bodies matching TypeScript interfaces in src/types/
